Setting Up The Library

Setting Ultimate Fancy Form is really easy, you just have to do a few things

1. Make sure you have jQuery

If you don't have jQuery in your project, no worries, UFF has it prepared, add this line in your head section

<script src="uff/jquery.js"></script>

2. Include jQuery UFF in your project

Add the following in your head section

<link href="uff/animate.css" rel="stylesheet" type="text/css">
<script src="uff/jquery-ultimate-fancy-form.min.js"></script>
<link href="uff/jquery-ultimate-fancy-form.css" rel="stylesheet" type="text/css">

3. Mark your form to use uff, by adding this class to it component-uff


4. Start Building.

I. Step Controller

The Step Controller is the CORE functionality of Ultimate Fancy Form

Inside your <form> you will have N amount of steps, these steps are marked with data-step

Step Usage Example

<form>
   <div data-step></div>
   <div data-step></div>
</form>

All the steps must be in the same container.

Moving Trough Steps

Moving From one step to another is easy, list of options

Attribute Name Attribute Role
data-step-next Display the next step
data-step-previous Display the previous step
data-step-display="%step_number%" Display a certain step, this is index based, so it starts from 0
data-step-finish This is the marker that tells Ultimate Fancy Form that this is the final step

Moving Trough Steps Example

<form>
   <div data-step>
     <a data-step-next>Next Step</a>
   </div>
   <div data-step>
     <a data-step-previous>Previous Step</a>
     <a data-step-next>Next Step</a>
   </div>
   <div data-step>
     <a data-step-previous>Previous Step</a>
     <a data-step-finish>Submit Form</a>
   </div>
</form>

II. Step Controller Mirror

This feature is something out of the box, there are few examples where you could use it.

All Step Mirrors must be in the same container.


Available Options

Attribute Name Attribute Role
data-step-mirror Display only this container on a certain step. For Example for data-step 3, only data-step-mirror 3 would be displayed
data-step-mirror-cascading Display this container on a certain step and the previous ones. For Example for data-step 3, this would display data-step-mirror-cascading 1 and 2 also

III. Input Mirror

Easily display the value of an input in a certain container

Attribute Name : data-input-mirror

Binding a Input Mirror By Name :

<span data-input-mirror='[name="input_name"]'></span>

Binding a Input Mirror By Class :

<span data-input-mirror=".input_class"></span>

Binding a Input Mirror By Id :

<span data-input-mirror="#input-id"></span>

VI. Elegant Sound

Add a sound to any element interaction, easily.

Adding a sound to an input :

<input name="name" data-elegant-sound="assets/sounds/sound-name.mp3"/>

Adding a sound to an input that plays on hover :

<input name="name" data-elegant-sound="assets/sounds/sound-name.mp3" data-elegant-sound-on="hover"/>

Adding a sound to an input that plays on mouseout of an area:

<div data-elegant-sound="assets/sounds/sound-name.mp3" data-elegant-sound-on="hover"></div>

Available Params :

Attribute Name Attribute Role Default
data-elegant-sound The sound mp3 file path -
data-elegant-sound-on The event/events you want play the song on click change keyup
data-elegant-sound-duration The duration of the sound in miliseconds 1000

XII. Validation

Validation is a minimalist validation module, it offers the option to display a certain container if an error occurs.

Making an input required - with error container ID identifier

<input name="name" data-validation-required="#input_name_required"/>
<div id="#input_name_required">This field is required<div>

Making an input required - with error container class identifier

<input name="name" data-validation-required=".input_name_required"/>
<div class="input_name_required">This field is required<div>

Available Validations

Alias Description
data-validation-required Making an input required
data-validation-email Making an input be validated if email

Tooltip Validation

Add the following in your head section

<link href="uff/hint.css" rel="stylesheet" type="text/css">

What's the difference ?

You need to add the following attribute to the inputs data-validation-tooltip

Inside your validation type, you're now going to put the displayed message instead of the container identifier :)

Making an input required - with tooltip

<input name="name" data-validation-tooltip data-validation-required="This field is required"/>

XIII. Input Replica

This is a new fancy feature, which allows you to have an input value in another input where needed, and you can also add validation rules

Replicating an input without any rules

<input name="name" />
<input name="name_replica" data-input-replica="[name='name']"/>

Replicating an input when a checkbox is checked

<input type="text" name="name" />
<input type="text" name="name_replica" data-input-replica="[name='name']" data-input-replica-on="[name='replicate_input_on']" />
<input type="checkbox" name="replicate_input_on" />

Available Params

Alias Description Type Default
data-input-replica Making an input required Required -
data-input-replica-on The input which tells the input replica when to replicate this certain input Optional -
data-input-replica-on-rule The data-input-replica-on object will be verified using .is(data-input-replica-on-rule) if it return true, the replication process will take place Optional :checked
